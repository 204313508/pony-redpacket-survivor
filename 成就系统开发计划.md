# 成就系统开发计划

## 📋 项目概述

为《小马红包大作战》添加完整的成就系统，包括10大类别、每类3个难度等级、共30个成就。成就数据存储在localStorage，界面参考图鉴设计风格。

---

## 🎯 核心功能清单

### 一、成就数据结构与存储（3个子功能）

#### 1.1 成就数据模型设计
- **实现方式**：创建成就配置对象（ACHIEVEMENT_CONFIG）
- **内容**：
  - 成就ID、名称、描述
  - 类别、难度等级（初级/中级/高级）
  - 完成条件（目标数值、触发类型）
  - 图标设计（emoji或Unicode字符）
  - 视觉特效（未获得/获得时/获得完成）
- **输出**：ACHIEVEMENT_CONFIG 常量对象
- **预估时间**：2小时

#### 1.2 成就状态管理类（AchievementManager）
- **实现方式**：创建 AchievementManager 类
- **功能**：
  - 初始化成就数据（从localStorage读取）
  - 保存成就数据到localStorage
  - 检查成就完成条件
  - 更新成就状态（未完成→已完成）
  - 触发成就解锁通知
- **输出**：AchievementManager 类
- **预估时间**：3小时

#### 1.3 成就存储与读取
- **实现方式**：localStorage API
- **功能**：
  - 键名：`redpacket_game_achievements`
  - 数据结构：JSON对象 `{achievementId: {unlocked: true, unlockedTime: timestamp}}`
  - 默认初始化：所有成就unlocked=false
- **输出**：存储/读取函数
- **预估时间**：1小时

### 二、成就界面UI设计（5个子功能）

#### 2.1 成就按钮添加（游戏开始界面）
- **实现方式**：修改 index.html 开始界面按钮行
- **位置**：在"开始游戏"和"图鉴"按钮旁边
- **样式**：奖杯图标🏆，圆形按钮，与图鉴按钮样式一致
- **功能**：点击打开成就界面
- **输出**：HTML按钮元素
- **预估时间**：0.5小时

#### 2.2 成就按钮添加（HUD界面）
- **实现方式**：修改 index.html HUD按钮组
- **位置**：在设置按钮⚙️旁边
- **样式**：奖杯图标🏆，圆形按钮，与设置按钮大小一致
- **功能**：点击打开成就界面（暂停游戏）
- **输出**：HTML按钮元素
- **预估时间**：0.5小时

#### 2.3 成就界面HTML结构
- **实现方式**：添加 achievementScreen div
- **参考**：bestiaryScreen 图鉴界面结构
- **内容**：
  - 标题：🏆 成就系统
  - 类别筛选标签（10个类别）
  - 成就网格容器
  - 成就详情面板（点击成就卡片后显示）
  - 关闭按钮
- **输出**：HTML结构
- **预估时间**：1.5小时

#### 2.4 成就卡片样式
- **实现方式**：CSS样式
- **设计**：
  - 网格布局（grid-template-columns: repeat(auto-fill, minmax(180px, 1fr))）
  - 卡片尺寸：约180x200px
  - 未获得状态：灰色背景，图标低透明度，描述模糊
  - 已获得状态：彩色渐变背景，图标发光，描述清晰
  - 悬停效果：放大、发光、边框高亮
  - 完成动画：解锁时闪烁+粒子特效
- **输出**：CSS样式类
- **预估时间**：2小时

#### 2.5 成就详情面板
- **实现方式**：动态内容更新
- **内容**：
  - 成就大图标
  - 成就名称、描述
  - 完成条件（当前值/目标值）
  - 完成时间
  - 进度条（未完成时显示）
  - 难度等级标识（⭐⭐⭐）
- **输出**：详情面板UI
- **预估时间**：1.5小时

### 三、成就逻辑与触发（4个子功能）

#### 3.1 游戏统计追踪器（GameStatsTracker）
- **实现方式**：创建 GameStatsTracker 类
- **功能**：
  - 追踪单局数据：击杀数、分数、等级、收集红包数、存活时间
  - 追踪全局数据：累计击杀、累计游戏局数、累计收集红包、最高分数
  - 追踪属性数据：最高攻击、最高防御、最高速度、最高生命
  - 追踪技能数据：各技能最高等级
  - 追踪Boss数据：击败Boss次数
  - 追踪天气数据：在不同天气下完成游戏次数
- **输出**：GameStatsTracker 类
- **预估时间**：3小时

#### 3.2 成就检查触发器
- **实现方式**：在游戏关键节点调用检查函数
- **触发点**：
  - 击杀怪物时 → 检查击杀类成就
  - 升级时 → 检查等级类成就
  - 收集红包时 → 检查收集类成就
  - 游戏结束时 → 检查分数、存活时间、天气类成就
  - 击败Boss时 → 检查Boss类成就
  - 属性提升时 → 检查属性类成就
  - 技能升级时 → 检查技能类成就
- **输出**：检查函数调用点
- **预估时间**：2小时

#### 3.3 成就解锁通知系统
- **实现方式**：创建 AchievementNotification 类
- **功能**：
  - 显示解锁提示框（屏幕中央或顶部）
  - 成就图标+名称动画效果
  - 音效播放（复用upgrade.mp3）
  - 自动消失（3秒后）
  - 队列处理（同时解锁多个成就时依次显示）
- **输出**：通知UI和逻辑
- **预估时间**：2小时

#### 3.4 成就进度更新
- **实现方式**：实时更新成就进度
- **功能**：
  - 更新成就详情面板的进度条
  - 更新成就卡片的进度显示
  - 未完成成就显示当前值/目标值
- **输出**：进度更新函数
- **预估时间**：1小时
